site-generator:generate-action-site

Full name:

org.nasdanika.html:site-generator-maven-plugin:2023.2.2:generate-action-site

Description:

Generates action site.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is not marked as thread-safe and thus does not support parallel builds.
  • Binds by default to the lifecycle phase: site.

Required Parameters

Name Type Since Description
<action> String - URI of the action from which to generate a site. Resolved relative to the project base directory
<pageTemplate> String - URI of the page template. Resolved relative to the project base directory

Optional Parameters

Name Type Since Description
<cleanWorkDir> boolean - If true the working directory is cleaned before generation
<contexts> List<String> - URL's of YAML or JSON resources which are loaded as contexts. if the URL ends with .yaml or .yml then it is treated as YAML, as a JSON object otherwise.
<diagnosticOutput> File - File to output diagnostic. If not set, diagnostic is output to the log.
<diagramGenerators> List<DiagramGenerator> - Pluggable diagram generators.
<drawioViewer> String - URL of the Drawio viewer script for rendering Drawio diagrams. Set if you are hosting your own Drawio site.
<errors> int - Number of known/expected errors. E.g. some blank pages or broken links. Build fails if the number of actual errors reported is different from this parameter.
<failStatuses> Status[] - Statuses on which to fail execution. FAIL if empty. Available statuses: FAIL, ERROR, WARNING, SUCCESS, INFO, CANCEL
<outputDirectory> File - Directory to output generated site
Default value is: target/action-site.
<progressOutput> File - File to output progress. If not set, progress is output to the log.
<semanticMaps> List<String> - URL's of YAML or JSON resources which are loaded as semantic maps of URI's to a map containing 'text', 'icon', 'tooltip', 'location' keys. Semantic maps are used to link model elements to externally defined element by URI. It is similar to how Java modules require other modules and then classes in that module may reference elements from the required modules by their fully qualified names. if the URL ends with .yaml or .yml then it is treated as YAML, as a JSON object otherwise.
<siteMapDomain> String - Site map domain/URL to output to sitemap.xml. Site map is not generated if this parameter is not set.
<workDirectory> File - Working directory for storing intermediate files
Default value is: target/action-site-work-dir.
