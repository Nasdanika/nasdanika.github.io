To generate a documentation website from a Drawio diagram and publish it to GitHub Pages, you can follow these steps using the Nasdanika CLI tools:

  1. Install Prerequisites: Make sure you have Java 17 or above and the Nasdanika CLI installed.

  2. Prepare Drawio Diagram: Ensure your .drawio file is ready, with diagram elements configured with documentation and necessary properties for site generation (such as documentation format, doc-ref, etc.).

  3. Generate HTML Application Model: Use the nsd drawio html-app command to generate an HTML application model from your diagram. Here is an example command:

    nsd drawio html-app -r root-action.yml --add-to-root site -r=-1 -F page-template.yml docs
    • -r root-action.yml: Specifies the root action configuration.
    • --add-to-root: Adds generated labels to the root action.
    • site: Sub-command to generate the documentation site.
    • -r=-1: Indicates to not fail on page errors during site generation.
    • -F page-template.yml: Defines the page template configuration.
    • docs: Specifies the output directory for the generated site.

Refer to the CLI commands documentation for more details.

  1. Configure GitHub Pages: Initialize a new Git repository (or use an existing one), and configure it to use the docs folder for GitHub Pages.

    git init
git remote add origin https://github.com/yourusername/yourrepository.git

  2. Commit and Push to GitHub:

    git add docs
git commit -m "Publish Drawio generated documentation"
git push -u origin main

  3. Activate GitHub Pages: Go to your GitHub repository settings and configure GitHub Pages to use the docs directory as the source.

  4. Access Your Site: Once GitHub Pages has been configured and published, you can access your documentation site via the provided URL like https://yourusername.github.io/yourrepository.
