IssueSeverity

Issue severity, e.g. Feature, Minor, Major, Crash.

Severity defines how the issue affects its product. Severity is different from priority. E.g. if an issue describes a situation where the product, e.g. a web application, crashes but its occurrence low because then severity would be Crash, but priority might be Low.
